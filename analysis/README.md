# Analysis

Turns the CSVs you collect into the tables and plots `docs/03` and `docs/04`
ask for: latency distributions, effective FPS vs drop rate, memory, the
cross-browser / WebGPU-vs-WASM comparison, and an output-quality tally.

## Setup

You need Python with a few libraries:

```bash
pip install jupyter pandas numpy matplotlib
```

(Your Mac already has Python via the conda `base` env, so `pip install ...`
then `jupyter notebook` should work directly.)

## Run it

```bash
cd analysis
jupyter notebook analysis.ipynb
```

Then Kernel → Restart & Run All. It runs out of the box against the bundled
**SAMPLE-** files so you can see what every chart looks like before you have
real data. Figures are written to `analysis/figures/`.

## Use it with YOUR data

1. **Session logs** — in the app, run your video end to end in each mode, click
   *Download CSV*, and drop the files into `analysis/data/`. They're named
   `fastvlm-<mode>-<timestamp>.csv`. Delete the `SAMPLE-` files (or leave them;
   the notebook reads every `fastvlm-*`/`SAMPLE-*` CSV except the two below).
   - Tip: if you test the same video at several sample intervals, rename the
     files so you can tell them apart, e.g. `fastvlm-caption-200ms.csv`.

2. **Harness results** — each time you run `runBenchmark()` in the browser
   console, copy the printed CSV line into `analysis/data/harness.csv` under the
   header, and add a `label` describing the run (e.g. `Chrome WebGPU q4`,
   `Chrome WASM (fallback)`). The notebook uses `label` for the x-axis and to
   compute the WebGPU-vs-WASM speedup.

3. Re-run all cells.

## What each section produces

| Section | Output | Goes in report as |
|---|---|---|
| 1–2 | Loads logs, drops the warm-up frame | (method note) |
| 3 | Latency table + bar chart by mode | Latency results |
| 4 | Latency histogram | Consistency / tail behavior |
| 5 | Effective FPS + drop-rate table & scatter | Frame-rate results |
| 6 | Harness latency (log scale) + memory + speedup | Cross-browser, WASM gap, memory |
| 7 | Output listing + optional hand-labeled quality tally | Quality results |
| 8 | Summary CSVs in `figures/` | Appendix |

## Files the notebook ignores

- `harness.csv` — read separately in Section 6, not as a session log.
- `quality_labels_template.csv` — generated by Section 7 for hand-labeling.

## The quality workflow (Section 7)

Latency says nothing about whether captions were *correct*. Section 7 prints the
raw outputs and writes `data/quality_labels_template.csv` with a blank
`quality` column. Fill each row with `good` / `vague` / `wrong`, re-run, and the
last cell tallies and charts accuracy by mode. This is the only metric that
needs your judgment — everything else is automatic.

## Reminder

Always keep the warm-up out of stats. The notebook drops `seq == 0` per file
automatically; if you compute anything by hand, do the same.
